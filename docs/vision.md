# Car Rental System — Vision

## Назва проєкту
Car Rental System

## Опис проблеми
Система автоматизує процес оренди автомобілів: реєстрацію клієнтів,
перевірку доступності авто, оформлення оренди, розрахунок вартості
та повернення транспортного засобу.

## Цільові користувачі
- Менеджер прокату — керує автопарком і орендами
- Клієнт — орендує автомобіль

## Основні сценарії використання
1. Клієнт реєструється в системі
2. Менеджер додає автомобіль до автопарку
3. Клієнт орендує доступний автомобіль на певний період
4. Клієнт повертає автомобіль, система розраховує підсумкову вартість
5. Менеджер переглядає список активних оренд

## Нефункціональні вимоги
1. Читабельність коду — чіткий поділ на шари, зрозумілі імена
2. Тестованість — бізнес-логіка відокремлена від UI та інфраструктури
3. Відновлення після помилки — некоректний ввід не ламає програму
4. Простота розширення — нові типи авто або тарифів без переписування

## Обмеження першої ітерації (Lab 34)
- Немає збереження у файл (тільки in-memory)
- Немає графічного інтерфейсу
- Один тип тарифу (фіксована ціна за день)
- Без автентифікації користувачів
