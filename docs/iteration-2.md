# Ітерація 2 — Lab 35

## Завершені use cases
1. Оренда авто з вибором тарифу (стандартний / зі знижкою)
2. Повернення авто з розрахунком вартості
3. Скасування активної оренди
4. Перегляд і фільтрація авто за ціною
5. Аналітика — загальний дохід, кількість авто і оренд

## Бізнес-правила
1. Авто не можна орендувати якщо воно недоступне
2. Дата кінця оренди повинна бути пізніше дати початку
3. Тільки активну оренду можна завершити або скасувати
4. Вартість розраховується стратегією на момент оренди
5. Дохід рахується тільки по завершених орендах

## Застосований патерн
**Strategy** — IPricingStrategy з реалізаціями StandardPricingStrategy і DiscountPricingStrategy.
Без цього патерну додавання нового тарифу потребувало б зміни RentalService.
На Lab 37 можна додати LongTermPricingStrategy без зміни існуючого коду.

## LINQ-запити
1. GetAvailableCars() — фільтрація за IsAvailable
2. GetCarsSortedByPrice() — сортування за ціною
3. GetCarsInPriceRange() — фільтрація за діапазоном
4. GetRentalsByClient() — оренди конкретного клієнта
5. GetTotalRevenue() — агрегація суми завершених оренд

## Спеціалізована колекція
Dictionary<string, IPricingStrategy> — швидкий пошук стратегії за назвою в O(1)

## Що змінилось відносно Lab 34
- Car, Client — додано optional Id в конструктор для десеріалізації
- Rental — додано PricingStrategyName, SetTotalCost()
- RentalService — додано CancelRental, LINQ-запити, підтримка стратегій
- Infrastructure — додано JsonCarRepository, JsonClientRepository, JsonRentalRepository
